# UX Laws Skill

This skill provides comprehensive knowledge of 30 core UX principles to help you design better user experiences.

## Core Knowledge

All 30 laws, including their definitions, key takeaways, and historical origins, are documented in the [catalog](references/catalog.md).

## How to Use This Skill

### 1. Identifying Relevant Laws
When a user presents a design problem or a user interface, use this skill to identify which UX laws are being applied correctly or which ones are being violated.
- **Example:** "The user is overwhelmed by these 50 menu items. According to Hick's Law, we should reduce the number of choices."

### 2. Explaining UX Principles
Provide clear, authoritative explanations for UX principles when asked.
- **Example:** "Jakob's Law states that users prefer your site to work like other sites they know because they spend most of their time elsewhere."

### 3. Guiding Design Decisions
Use the key takeaways from each law to propose specific design improvements.
- **Example:** "To leverage the Von Restorff Effect, let's make the 'Call to Action' button a contrasting color so it stands out from the rest of the page."

### 4. Gestalt Principles for Grouping
If the user asks about grouping elements, refer to the following Gestalt laws in the catalog:
- Law of Common Region
- Law of Proximity
- Law of Similarity
- Law of Uniform Connectedness

### 5. Memory and Cognitive Load
For issues related to user memory or overwhelming interfaces, refer to:
- Chunking
- Cognitive Load
- Miller's Law
- Working Memory
- Serial Position Effect

## Workflow
1. **Analyze:** Identify the UX challenge (e.g., choice overload, poor memorability).
2. **Reference:** Consult [catalog.md](references/catalog.md) for the most relevant law(s).
3. **Recommend:** Provide the definition and specific "Key Takeaways" to justify your design recommendations.
